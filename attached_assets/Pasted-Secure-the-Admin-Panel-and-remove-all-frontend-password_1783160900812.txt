Secure the Admin Panel and remove all frontend password storage.

SECURITY REQUIREMENTS

- Never store admin passwords in HTML, CSS, JavaScript, localStorage, sessionStorage, cookies, or any client-side code.
- Admin credentials must never be visible in page source, browser developer tools, network requests, or downloadable files.

BACKEND AUTHENTICATION

- Create a secure backend authentication system.
- Store passwords only as salted and hashed values.
- Use secure password hashing.
- Verify passwords only on the server side.

ADMIN LOGIN

- Create a hidden admin login page.
- Require username and password.
- Use secure server-side authentication.
- Redirect unauthorized users away from admin pages.

ACCESS CONTROL

- Only authenticated admins can:
  - Add Games
  - Edit Games
  - Delete Games
  - Change Prices
  - Change Posters
  - Modify Featured Deals

- Block direct access to admin routes without authentication.

SESSION SECURITY

- Create secure login sessions.
- Auto logout after inactivity.
- Secure logout button.
- Prevent session hijacking.
- Prevent unauthorized access after logout.

PROTECTION

- Add brute-force protection.
- Add rate limiting for login attempts.
- Lock account temporarily after multiple failed logins.
- Prevent SQL injection.
- Prevent XSS attacks.
- Prevent CSRF attacks.
